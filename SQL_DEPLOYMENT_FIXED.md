# AuraGrade SQL Deployment - FIXED VERSION

## What Was Wrong
The original schema.sql had a PostgreSQL syntax error:
```
ERROR: 42601: syntax error at or near "NOT" LINE 10: CREATE TYPE IF NOT EXISTS prof_status_enum...
```

PostgreSQL **does NOT support** `IF NOT EXISTS` with `CREATE TYPE` for ENUM types.

## What's Fixed
Both ENUM types now use DO blocks with exception handling:
```sql
DO $$ BEGIN
  CREATE TYPE prof_status_enum AS ENUM ('Pending', 'Approved', 'Overridden', 'Flagged', 'Audited');
EXCEPTION WHEN duplicate_object THEN NULL;
END $$;
```

This pattern:
- ✅ Creates the ENUM if it doesn't exist
- ✅ Silently skips if it already exists
- ✅ Works with PostgreSQL (Supabase)

-- ============================================================

-- ── GAP 1: exception_queue has RLS enabled but ZERO policies ──────────────
-- Staff can read exception queue entries
DO $$ BEGIN
  IF NOT EXISTS (
    SELECT 1 FROM pg_policies
    WHERE tablename = 'exception_queue' AND policyname = 'Staff read exception queue'
  ) THEN
    CREATE POLICY "Staff read exception queue" ON exception_queue
      FOR SELECT USING (
        auth.uid() IN (
          SELECT id FROM profiles WHERE role IN ('ADMIN_COE', 'HOD_AUDITOR', 'EVALUATOR')
        )
      );
  END IF;
END $$;

-- Staff can update exception queue entries (resolve/reject)
DO $$ BEGIN
  IF NOT EXISTS (
    SELECT 1 FROM pg_policies
    WHERE tablename = 'exception_queue' AND policyname = 'Staff update exception queue'
  ) THEN
    CREATE POLICY "Staff update exception queue" ON exception_queue
      FOR UPDATE USING (
        auth.uid() IN (
          SELECT id FROM profiles WHERE role IN ('ADMIN_COE', 'HOD_AUDITOR', 'EVALUATOR')
        )
      );
  END IF;
END $$;

-- Service role (backend) can insert into exception queue
DO $$ BEGIN
  IF NOT EXISTS (
    SELECT 1 FROM pg_policies
    WHERE tablename = 'exception_queue' AND policyname = 'Backend insert exception queue'
  ) THEN
    CREATE POLICY "Backend insert exception queue" ON exception_queue
      FOR INSERT WITH CHECK (true);
  END IF;
END $$;

## Step 3: Run batch_jobs_migration.sql

**Copy-paste this after Step 2 succeeds:**

```sql
-- ============================================================
-- AuraGrade — In-Memory State Migration
-- Moves batch job tracking from process memory to Supabase
-- Run in Supabase SQL Editor
-- ============================================================

-- ── Table: batch_jobs ─────────────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS batch_jobs (
    id              TEXT PRIMARY KEY,              -- short UUID (12 chars)
    status          TEXT NOT NULL DEFAULT 'processing',
                    -- processing | completed | failed
    total_pages     INT NOT NULL DEFAULT 0,
    processed_pages INT NOT NULL DEFAULT 0,
    errors          JSONB DEFAULT '[]'::jsonb,
    results         JSONB DEFAULT '[]'::jsonb,     -- per-page grading results
    aggregated_result JSONB,                       -- final merged result
    detected_student  JSONB,                       -- header-parser auto-detect
    created_by      UUID REFERENCES auth.users(id) ON DELETE SET NULL,
    created_at      TIMESTAMPTZ DEFAULT now(),
    completed_at    TIMESTAMPTZ,
    expires_at      TIMESTAMPTZ DEFAULT (now() + INTERVAL '24 hours')
                    -- auto-cleanup: batch jobs older than 24h are stale
);

CREATE INDEX IF NOT EXISTS idx_batch_jobs_status    ON batch_jobs (status);
CREATE INDEX IF NOT EXISTS idx_batch_jobs_created   ON batch_jobs (created_at DESC);
CREATE INDEX IF NOT EXISTS idx_batch_jobs_expires   ON batch_jobs (expires_at);
CREATE INDEX IF NOT EXISTS idx_batch_jobs_created_by ON batch_jobs (created_by);

ALTER TABLE batch_jobs ENABLE ROW LEVEL SECURITY;

-- Users can only see their own batch jobs
CREATE POLICY "Users read own batch jobs" ON batch_jobs
  FOR SELECT USING (created_by = auth.uid());

-- Staff can see all batch jobs
CREATE POLICY "Staff read all batch jobs" ON batch_jobs
  FOR SELECT USING (
    auth.uid() IN (
      SELECT id FROM profiles
      WHERE role IN ('ADMIN_COE', 'HOD_AUDITOR', 'EVALUATOR')
    )
  );

-- Backend inserts/updates batch jobs
CREATE POLICY "Backend manage batch jobs" ON batch_jobs
  FOR ALL WITH CHECK (true);
```
